# Протокол испытаний (нагрузочное тестирование)

## Объект испытаний

- Сервис: `go-service-ci`
- Endpoint: `GET /api/v1/satellite/moon`
- Локальный адрес: `http://localhost:8082`

## Инструмент

- Apache JMeter 5.6.3
- Сценарий: `load_tests/Summary Report.jmx`

## Условия запуска

- Локальный стенд поднят через:
  - `docker compose up -d --build`
- База данных и Redis доступны
- В БД есть тестовые данные для `moon`

## Профиль нагрузки

- Количество потоков (users): `50`
- Ramp-up: `10s`
- Общее количество запросов: `5000`
- Проверка ответа: HTTP `200`

## Результаты

- Samples: `5000`
- Error %: `0.00%`
- Throughput: `~503.6 req/sec`
- Average: `~2 ms`
- p95: `~4 ms`
- p99: `~8 ms`
- Max: `~161 ms`

## Вывод

- Сервис корректно обработал нагрузку без ошибок.
- При текущем профиле тестирования latency находится на стабильном уровне.
- Для дальнейшей оценки производительности рекомендуется добавить stress-профиль (большее число пользователей и длительность прогона).
